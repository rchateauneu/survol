SURVOL AUDIENCE AND USERS

Survol has several classes of users, working on the same systems and concepts, from different perspectives.

Consultant

User is a management service consultant or a managed services provider: Survol is an investigation tool.

A consultant working for a management service, on an existing system, starts her/his mission with an assessment stage where the client application is evaluated. The goal is to define options and solutions and create a roadmap for future enhancement. The consultant immediately needs, as much information as possible about the internals, structure, etc.… of the legacy application. Very simple information might be difficult to obtain if the technical experts are not available. However, it is of utmost important to concentrate on the high-level functionalities of the client's application. Survol provides architectural and technical insight about any IS: Qualitative information : client/server structure, components, packages and libraries dependencies etc… and quantitative metrics: Code and data size, transfer volumes, coupling factor between modules etc. and many other information given by Survol analysers and extensions. These metrics and qualitative information are used by the Consultant to create her/his assessment result, found out which solutions are feasible. Software developer AND TECHNICAL LEAD

For a software developer in charge of maintenance or enhancing existing applications, Survol is an investigation tool. It runs commands which are commonly used by developers wishing to understand the internal behavior of an application. What Survol adds is:

  • The command output is in several formats: Text, RDF, JSON etc...
  • With a graphic output, the result can be shown to non-technical users, to show a task progress
  • Several commands'results can be easily merged, and the execution reproduced.
